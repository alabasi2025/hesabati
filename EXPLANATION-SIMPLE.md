
## 🔧 كيف ننفذ الحل؟

### خطوة بسيطة جداً:

#### 1️⃣ في Backend - إضافة endpoint جديد:

**الملف:** `backend/src/routes/api/accounts.routes.ts`

```typescript
// إضافة هذا:
accountsRoutes.get('/businesses/:bizId/custody-accounts', 
  bizAuthMiddleware(), 
  safeHandler('جلب حسابات العهد', async (c) => {
    const bizId = getBizId(c);
    
    // جلب حسابات العهد فقط
    const rows = await db
      .select()
      .from(accounts)
      .innerJoin(accountSubNatures, 
        eq(accounts.accountSubNatureId, accountSubNatures.id))
      .where(and(
        eq(accounts.businessId, bizId),
        eq(accountSubNatures.natureKey, 'custody')
      ));
    
    return c.json(rows); // 2 حساب فقط!
  })
);
```

#### 2️⃣ في Frontend - استخدام الـ endpoint الجديد:

**الملف:** `frontend/src/app/services/api.service.ts`

```typescript
// إضافة هذا:
getCustodyAccounts(bizId: number) {
  return this.request<any[]>(`/businesses/${bizId}/custody-accounts`);
}
```

**الملف:** `frontend/src/app/pages/custody/custody.ts`

```typescript
// تغيير من:
```

## 🎯 اقتراحات أخرى مهمة

### 2️⃣ ربط custody_records بالحسابات

**المشكلة الحالية:**
```
لديك:
  - حسابات عهدة: CUS-01, CUS-02 (في accounts)
  - سجلات عهد: (في custody_records)

لكنها غير مرتبطة! 😕
```

**الحل:**
```sql
-- إضافة عمود للربط
ALTER TABLE custody_records 
ADD COLUMN account_id INTEGER REFERENCES accounts(id);

-- الآن يمكن:
سجل عهدة "محمد" → مرتبط بـ CUS-01
سجل عهدة "أحمد" → مرتبط بـ CUS-02
```

**الفائدة:**
- تعرف أي سجل يخص أي حساب
- تتبع الأرصدة بشكل أفضل
- مثل funds/suppliers/warehouses
