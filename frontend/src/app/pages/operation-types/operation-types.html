auto_awesome القالب = عملية مقيدة بالكامل

المستخدم بس يختار القالب ويعبي المبلغ ويحفظ. كل شي محدد مسبقاً: نوع العملية، الحساب الرئيسي، والحسابات المقابلة.

format_list_numbered خطوات إنشاء القالب

1 نوع العملية: سند قبض / سند صرف / قيد محاسبي
2 طريقة الدفع: نقداً / بنك / صراف / محفظة (للسندات فقط)
3 الحساب الرئيسي: الصندوق أو البنك اللي يستلم أو يصرف
4 الحسابات المقابلة: حسابات الموظفين أو الأنظمة المرتبطة
5 الشاشات: وين يظهر القالب (تحصيل، توريد، سندات، قيود)

rule قواعد مهمة

سند قبض/صرف: الحساب الرئيسي لازم يكون صندوق أو بنك أو صراف أو محفظة.
قيد محاسبي: لما الحسابات ما تكون صناديق (مثل تحويل بين حسابات فوترة).
نفس القالب ممكن يظهر في عدة شاشات.
