## 🎯 اقتراحات عاجلة (أولوية عالية)

### 1️⃣ **تحسين الأداء - Endpoints مخصصة**

**المشكلة الحالية:**
صفحات مثل العهد، الوسيطة تحمّل **جميع** الحسابات (159 حساب) ثم تفلتر في Frontend.

**الحل المقترح:**
إضافة endpoints مخصصة:

```typescript
// Backend
GET /api/businesses/:bizId/custody-accounts      // 2 حساب فقط
GET /api/businesses/:bizId/intermediary-accounts // 3 حسابات فقط
GET /api/businesses/:bizId/pending-accounts      // 3 حسابات فقط
```

**الفائدة:**
- ⚡ تحميل أسرع
- 📉 تقليل استهلاك الـ bandwidth
- ✨ تجربة أفضل

**الجهد:** ساعة واحدة

---

### 2️⃣ **ربط custody_records بالحسابات**

**المشكلة:**
حالياً `custody_records` منفصلة تماماً عن حسابات العهد.

**الحل المقترح:**
إضافة `accountId` في `custody_records`:

```sql
ALTER TABLE custody_records 
ADD COLUMN account_id INTEGER REFERENCES accounts(id);
```

**الفائدة:**
- 🔗 ربط واضح بين السجل والحساب
- 📊 تتبع أفضل للأرصدة
- ✅ توحيد مع funds/suppliers/warehouses

**الجهد:** 30 دقيقة

---

### 3️⃣ **إكمال النوعين المتبقيين**

**الأنواع بدون بيانات:**
- `budget` (ميزانية) - BDG
- `settlement` (تصفية) - STL

**الحل المقترح:**
إنشاء بيانات أولية:

```typescript
// seed.ts
// حسابات الميزانية
BDG-01: ميزانية التشغيل السنوية
BDG-02: ميزانية الصيانة
BDG-03: ميزانية الرواتب

// حسابات التصفية
STL-01: تصفية حساب قديم
STL-02: تصفية شراكة منتهية
```

**الفائدة:**
- ✅ **14/14 نوع** لها بيانات
- 📊 أمثلة واضحة للاستخدام

**الجهد:** 15 دقيقة
