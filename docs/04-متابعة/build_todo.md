## المرحلة 1: الحسابات (مكتمل ✅)

- [x] صفحة حسابات موحدة تدمج: الحسابات العادية + الصناديق + حسابات الفوترة

- [x] نوع إضافي: فوترة (billing) — من جدول employee_billing_accounts
- [x] فلاتر ديناميكية حسب النوع مع فلاتر فرعية (نوع الصندوق، البنك، النظام، إلخ)
- [x] فلتر حسب المحطة (للصناديق والفوترة)
- [x] CRUD كامل (إضافة، تعديل، حذف)
- [x] أيقونة "مبدأ العمل" + شرح الصفحة
- [x] بيانات الحساب: الاسم، النوع، رقم الحساب، الجهة، المسؤول، ملاحظات

### 1.2 ربط الحسابات (تغيرت الآلية ✅)
> **ملاحظة مهمة:** تم نقل آلية ربط الحسابات من صفحة الحسابات إلى صفحة أنواع العمليات (القوالب). الربط يتم الآن عند إنشاء القالب:
> - الخطوة 4 في الـ wizard: اختيار المصدر (الطرف الأول) — أي صندوق أو بنك أو صراف أو محفظة بالتحديد
> - الخطوة 5 في الـ wizard: اختيار الحسابات المرتبطة (الطرف الثاني) — من كل الحسابات الموجودة في صفحة الحسابات
> - القالب هو اللي يحدد مين يتعامل مع مين، مش الحساب نفسه

## ملاحظات عامة

- **كل صفحة لازم فيها أيقونة "مبدأ العمل"** مع شرح كامل للمستخدم
- **كل ما نحدث صفحة نحدث الشرح معها**
- **كل عملية فيها خيار إضافة مرفق** (صور، سندات، إشعارات)
- **ربط الحسابات يتم عبر القوالب** في صفحة أنواع العمليات — القالب يحدد الطرف الأول والطرف الثاني
- **النظام مرن وقابل للتعديل** — يتطور مع التصفيات
