## مبادئ التصميم والبناء

**النظام عملي وليس محاسبي تقليدي.** المسميات والشاشات مصممة حسب طريقة عمل المالك الفعلية، وليس حسب المصطلحات المحاسبية الرسمية. الهدف أن أي موظف (مدير محطة أو متحصل أو محاسب) يقدر يستخدم النظام بدون خلفية محاسبية.

**نظام أنواع العمليات (القوالب):** بدلاً من صلاحيات محاسبية معقدة، كل عملية متكررة يتم إنشاء "نوع" لها (قالب جاهز) فيه الحسابات المرتبطة والإعدادات. المستخدم يفتح النوع ويعبي الأرقام فقط.

**أيقونة "مبدأ العمل":** كل صفحة وشاشة فيها أيقونة شرح توضح للمستخدم كيف يشتغل في هذي الصفحة. تتحدث مع كل تطوير.

**المرفقات:** كل عملية فيها خيار إضافة مرفق (صورة سند، إشعار تحويل، كشف بنك...) للتوثيق.

**صفحات وشاشات تحاكي طبيعة كل عمل:** النظام متعدد الأعمال (محطات كهرباء، وأعمال أخرى مختلفة). كل عمل له صفحاته وشاشاته الخاصة المبنية حسب طبيعته - مش صفحات عامة تنطبق على الكل. المحطات لها صفحات تحصيل وتوريد وفوترة، وعمل ثاني ممكن يكون له صفحات مختلفة تماماً. المشترك بين الأعمال قليل (الأساسيات فقط). التصميم يفتهم لك من نفسه.

**مرونة:** النظام قابل للتعديل والتطوير - يتطور مع تصفية الحسابات المتراكمة وتقسيم الأعمال.
