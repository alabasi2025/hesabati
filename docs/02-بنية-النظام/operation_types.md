# أنواع العمليات (Operation Types)

> **الغرض:** توثيق كل أنواع العمليات (القوالب) في النظام. كل نوع عملية يحدد الفئة (أين يظهر) والحسابات المرتبطة (الصلاحيات).
> **آلية التحديث:** نقرأ الملف ونعيد كتابة القسم المعني فقط مع التحديث في مكانه الصحيح.
> آخر تحديث: 27 فبراير 2026

---

## آلية القالب (المبدأ الأساسي)

> **القالب = عملية مقيدة بالكامل.** المستخدم بس يختار القالب ويعبي المبلغ ويحفظ.

**عند إنشاء قالب جديد يُحدد:**

| الخطوة | البند | التفاصيل |
|---|---|---|
| 1 | **نوع العملية** | سند قبض / سند صرف / قيد محاسبي (مش عشوائي - يحدد طبيعة القالب) |
| 2 | **طريقة القبض/الصرف** | نقداً / بنك / صراف / محفظة إلكترونية (تظهر فقط للسندات) |
| 3 | **الحساب الرئيسي** | الحساب اللي يستلم (قبض) أو يصرف منه (صرف). **قاعدة:** سند قبض/صرف → الحساب الرئيسي لازم يكون صندوق أو بنك أو صراف أو محفظة فقط. قيد → أي نوع حساب. |
| 4 | **الحسابات المقابلة** | نختار نوع الحساب أول (فوترة، صندوق، صراف...) → تظهر الحسابات من هذا النوع → نختار منها. ممكن نضيف حسابات من أنواع مختلفة. |
| 5 | **الشاشات** | نحدد في أي شاشات يظهر القالب (ممكن يظهر في أكثر من شاشة) |

**قواعد مهمة:**
- سند قبض → الحساب الرئيسي لازم صندوق/بنك/صراف/محفظة (مستحيل يكون حساب فوترة أو محاسبي)
- سند صرف → نفس الشي - الحساب الرئيسي لازم صندوق/بنك/صراف/محفظة
- قيد محاسبي → لما الحسابات ما تكون صناديق أو بنوك (مثل تحويل بين حسابات فوترة)
- القالب مقيد بالكامل - الحسابات محددة مسبقاً والمستخدم ما يحتاج يختار وين يروح
- نفس القالب ممكن يتكرر في عدة شاشات

| الفئة | الكود | أين يظهر القالب | ملاحظات |
