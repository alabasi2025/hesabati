# توثيق API (حساباتي)

## المواصفة الرسمية — OpenAPI 3.0

المواصفة الرسمية للـ API موجودة في:

- **الملف:** [openapi.yaml](./openapi.yaml)

هذا الملف يصف المسارات والطرق والأجسام والاستجابات والمصادقة (JWT) حسب التنفيذ الفعلي في الـ Backend.

### استخدام المواصفة

- **عرض تفاعلي:** استخدم أداة مثل [Swagger UI](https://swagger.io/tools/swagger-ui/) أو [Redoc](https://redocly.com/redoc/) وافتح `docs/openapi.yaml` لعرض الـ API في المتصفح.
- **التحقق الآلي:** يمكن استخدام المواصفة للتحقق من تطابق الواجهة الأمامية مع الـ API (اختبارات تعاقدية).
- **توليد العميل:** أدوات مثل `openapi-generator` أو `@openapi-typescript-codegen` يمكن أن تولد عميل TypeScript من `openapi.yaml`.

### نطاق التوثيق الحالي

- **مُوثَّق:** المصادقة (تسجيل الدخول، التسجيل، /me)، الأعمال، المحطات، الحسابات، الصناديق، السندات (جلب/إضافة/حذف)، لوحة التحكم، نقاط الصحة.
- **المسارات الكاملة:** جميع المسارات الفعلية موجودة في الكود في `backend/src/routes`. يمكن توسيع `openapi.yaml` ليشمل المزيد من المسارات (الشركاء، الموردين، المخازن، التصفيات، الرواتب، القيود، إلخ) عند الحاجة.

### الخادم الافتراضي

في المواصفة، الخادم الافتراضي هو `http://localhost:3000`. في بيئة أخرى غيّر قيمة `servers[0].url` أو استخدم المتغيرات المناسبة.

### المصادقة

جميع المسارات المحمية تتطلب رأس:

```
Authorization: Bearer <JWT>
```

يتم الحصول على الـ JWT من `POST /api/auth/login`.
