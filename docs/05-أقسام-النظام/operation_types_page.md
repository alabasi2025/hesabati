# صفحة أنواع العمليات (Operation Types)

## الغرض

صفحة أنواع العمليات هي المكان المركزي لإنشاء وإدارة **أنواع القوالب** و**القوالب**. القوالب هي عمليات جاهزة تسهّل على المستخدم تنفيذ العمليات بسرعة ودقة.

> **ملاحظة مستقبلية:** حالياً النظام يدعم القالب المالي فقط. لاحقاً سيتم التوسع لدعم أنواع أخرى (مخزني، مخزني مالي، وغيرها).

---

## المستوى الأول: أنواع القوالب (التصنيفات)

المستخدم ينشئ أنواع القوالب يدوياً من الشاشة نفسها. أنواع القوالب هي التصنيفات الرئيسية التي تُجمع تحتها القوالب.

**أمثلة على أنواع القوالب:**

| نوع القالب | الوصف |
|-----------|-------|
| تحصيل | قوالب التحصيل اليومي من المحطات |
| توريد | قوالب التوريد للصرافين والمحافظ |
| سندات | قوالب سندات القبض والصرف |
| قيود | قوالب القيود المحاسبية |

> **ملاحظة:** أنواع القوالب ليست ثابتة في الكود — المستخدم ينشئها ويسميها حسب طبيعة عمله.

---

## المستوى الثاني: القوالب (المالية)

تحت كل نوع قالب، ينشئ المستخدم القوالب الفعلية.

### حقول القالب المالي

| الحقل | الوصف |
|-------|-------|
| **الاسم** | اسم القالب (مثلاً: تحصيل يومي الدهمية) |
| **نوع القالب** | التصنيف الذي ينتمي إليه (ينشئه المستخدم) |
| **نوع السند** | قبض / صرف / قيد |
| **طريقة الدفع** | نقداً / بنك / صراف / محفظة (فقط للقبض والصرف) |
| **الوصف** | وصف اختياري |
| **الأيقونة** | أيقونة البطاقة |
| **اللون** | لون البطاقة |

---

## سلوك اختيار الحسابات حسب طريقة الدفع

### عند اختيار قبض أو صرف:

يظهر حقل **طريقة الدفع** ويتغير الحساب المعروض حسب الاختيار:

| طريقة الدفع | الحسابات التي تظهر |
|-------------|-------------------|
| **نقداً** (cash) | الصناديق فقط |
| **بنك** (bank) | البنوك فقط |
| **صراف** (exchange) | الصرافين فقط |
| **محفظة إلكترونية** (e_wallet) | المحافظ الإلكترونية فقط |

### عند اختيار قيد:

لا يظهر حقل طريقة الدفع. تظهر **حسابات دليل الحسابات** مباشرة لاختيار الحسابات المدينة والدائنة.

---

## الحسابات المرتبطة بالقالب (Linked Accounts)

كل قالب يمكن ربطه بعدة حسابات. هذه الحسابات هي التي تظهر للمستخدم عند استخدام القالب لتنفيذ عملية.

| الحقل | الوصف |
|-------|-------|
| **الحساب** | حساب عادي أو حساب فوترة موظف |
| **التسمية** | اسم الحساب كما يظهر في القالب |
| **الصلاحية** | قبض فقط / صرف فقط / الاثنين |
| **الترتيب** | ترتيب ظهور الحساب |

---

## البنية في قاعدة البيانات

### جدول `operation_types`

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | serial | المعرف |
| business_id | integer | ربط بالعمل |
| name | varchar(200) | اسم القالب |
| description | text | وصف اختياري |
| icon | varchar(50) | أيقونة |
| color | varchar(20) | لون |
| category | varchar(50) | نوع القالب - تصنيف ينشئه المستخدم يدوياً |
| voucher_type | varchar(30) | نوع السند (payment/receipt/journal) |
| payment_method | varchar(30) | طريقة الدفع (cash/bank/exchange/e_wallet) - للقبض والصرف فقط |
| screens | text | الشاشات المخصصة التي يظهر فيها |
| requires_attachment | boolean | يتطلب مرفق |
| has_multi_lines | boolean | يدعم أسطر متعددة |
| sort_order | integer | ترتيب العرض |
| is_active | boolean | نشط |

### جدول `operation_type_accounts`

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | serial | المعرف |
| operation_type_id | integer | ربط بالقالب |
| account_id | integer | حساب عادي |
| employee_billing_account_id | integer | حساب فوترة موظف |
| label | varchar(200) | تسمية الحساب |
| permission | varchar(30) | الصلاحية (both/receive_only/pay_only) |
| sort_order | integer | الترتيب |

---

## الـ API Endpoints

| Endpoint | الوظيفة |
|----------|---------|
| GET /businesses/:bizId/operation-types | جلب كل القوالب (مع فلتر حسب category أو screen) |
| GET /operation-types/:id | جلب قالب واحد بالتفصيل |
| POST /businesses/:bizId/operation-types | إنشاء قالب جديد مع حساباته |
| PUT /businesses/:bizId/operation-types/:id | تعديل قالب |
| DELETE /businesses/:bizId/operation-types/:id | حذف قالب وحساباته |

---

## ملاحظات

- تم حذف حقلي `main_account_id` و `main_fund_id` لأن طريقة الدفع هي التي تحدد الحساب (صندوق/بنك/صراف/محفظة).
- حقل `category` يجب أن يكون ديناميكياً (ينشئه المستخدم) وليس قائمة ثابتة في الكود.
- عند اختيار طريقة الدفع، يجب فلترة الحسابات المعروضة حسب النوع المناسب.
- القوالب تُستخدم لاحقاً داخل الشاشات المخصصة لتنفيذ العمليات.
