# الشاشات المخصصة (Custom Screens)

## الغرض

الشاشات المخصصة هي لوحات عمل ينشئها المستخدم يدوياً حسب طبيعة عمله. تعمل بنظام **Widget-Based Dashboard Builder** — يعني بناء الشاشات من عناصر (ويدجت) قابلة للسحب والإفلات والتكبير والتصغير والتخصيص الكامل.

---

## الآلية الاحترافية

### المستوى الأول: مكتبة العناصر (Widgets Library)

مكتبة مركزية فيها عناصر جاهزة، كل عنصر يؤدي وظيفة واحدة:

| العنصر (Widget) | الوظيفة | الوصف |
|----------------|---------|-------|
| **عنصر القوالب** | تنفيذ العمليات | يعرض قوالب العمليات المختارة كبطاقات، الضغط عليها يفتح نموذج التنفيذ |
| **عنصر السجل** | عرض الحركات | يعرض سجل العمليات المنفذة مع فلاتر (تاريخ، نوع، مبلغ) |
| **عنصر مراقبة الحسابات** | مراقبة الأرصدة | يعرض حسابات مختارة مع أرصدتها وآخر حركاتها |
| **عنصر الإحصائيات** | أرقام ملخصة | يعرض KPIs (إجمالي التحصيل، إجمالي الصرف، عدد العمليات) |
| **عنصر الرسم البياني** | تحليل بصري | رسوم بيانية للحركات والأرصدة |
| **عنصر الملاحظات** | ملاحظات نصية | ملاحظات وتذكيرات |

> **ملاحظة:** المكتبة قابلة للتوسيع — كل ما نطور العمل نضيف عناصر جديدة.

---

### المستوى الثاني: قوالب الشاشات (Screen Templates)

قوالب جاهزة فيها عناصر مرتبة بتصميم معين. تسرّع إنشاء الشاشات:

| قالب الشاشة | العناصر المضمنة | الاستخدام |
|-------------|----------------|----------|
| **قالب تحصيل** | قوالب عمليات + سجل + مراقبة حسابات | شاشات التحصيل اليومي |
| **قالب توريد** | قوالب عمليات + سجل + إحصائيات | شاشات التوريد |
| **قالب مراقبة** | مراقبة حسابات كبير + إحصائيات + رسم بياني | شاشات المراقبة والمتابعة |
| **قالب تقارير** | رسم بياني + إحصائيات + سجل | شاشات التقارير |
| **قالب فارغ** | لا شيء | بناء من الصفر |

> **ملاحظة:** قوالب الشاشات قابلة للتوسيع — كل ما نطور العمل نضيف قوالب شاشات جديدة.

---

### المستوى الثالث: إنشاء وتخصيص الشاشة

عند إنشاء شاشة مخصصة جديدة:

**الخطوة 1:** اختيار القاعدة
- اختيار قالب شاشة جاهز كقاعدة
- أو البدء من شاشة فارغة

**الخطوة 2:** التشكيل والتصميم
- سحب عناصر من المكتبة وإفلاتها في الشاشة (Drag & Drop)
- نسخ عناصر من شاشات أخرى
- إضافة عناصر من قوالب شاشات مختلفة ودمجها
- حذف عناصر لا يحتاجها المستخدم

**الخطوة 3:** التخصيص
- تغيير حجم كل عنصر (تكبير / تصغير)
- تغيير موضع كل عنصر (سحب وإعادة ترتيب)
- تغيير ألوان كل عنصر
- تغيير مسميات كل عنصر
- إخفاء / إظهار عناصر

**الخطوة 4:** ربط المحتوى
- اختيار القوالب (من أنواع العمليات) لعنصر القوالب
- اختيار الحسابات لعنصر المراقبة
- تحديد الفلاتر لعنصر السجل

## إعدادات التبويب الجانبي

الشاشات المخصصة تُدار من صفحة **إعدادات التبويب الجانبي** الموجودة حالياً في النظام:

| التبويب | الوظيفة |
|---------|---------|
| **الأقسام** | إدارة أقسام القائمة الجانبية |
| **الشاشات** | إنشاء وإدارة الشاشات المخصصة |
| **صلاحيات المستخدمين** | تحديد أي شاشات تظهر لكل مستخدم |

---

## صلاحيات الشاشة

من تبويب **صلاحيات المستخدمين**، المدير يحدد لكل مستخدم:
- أي شاشات تظهر له
- ترتيب الشاشات
- صلاحيات التنفيذ (عرض فقط / تنفيذ عمليات)

---

## المكتبات المطلوبة

### 1. السحب والإفلات وتخطيط الشبكة

| المكتبة | الغرض | الرابط |
|---------|-------|--------|
| **angular-gridster2** | تخطيط الشبكة مع سحب وإفلات وتغيير الحجم — الأنسب لبناء الداشبورد | [npm](https://www.npmjs.com/package/angular-gridster2) |
| **@angular/cdk/drag-drop** | سحب وإفلات أساسي من Angular CDK — للعناصر البسيطة | مضمن في Angular CDK |
| **@katoid/angular-grid-layout** | بديل خفيف لـ gridster2 — تخطيط شبكة تفاعلي | [npm](https://www.npmjs.com/package/@katoid/angular-grid-layout) |

> **التوصية:** استخدام **angular-gridster2** كمكتبة رئيسية لأنها الأكثر نضجاً ودعماً لـ Angular وتدعم: سحب وإفلات، تغيير الحجم، تخطيط تلقائي، حفظ واستعادة التخطيط.

### 2. الحركات والانتقالات (Animations)

| المكتبة | الغرض | الرابط |
|---------|-------|--------|
| **GSAP (GreenSock)** | حركات سلسة واحترافية — الأفضل أداءً | [gsap.com](https://gsap.com/) |
| **@angular/animations** | حركات Angular الأصلية — للانتقالات البسيطة | مضمن في Angular |
| **animate.css** | حركات CSS جاهزة — سهلة الاستخدام | [npm](https://www.npmjs.com/package/animate.css) |

> **التوصية:** استخدام **GSAP** للحركات المعقدة (فتح/إغلاق العناصر، انتقالات الصفحات) و **@angular/animations** للحركات البسيطة (hover، fade).

### 3. التأثيرات ثلاثية الأبعاد (3D Effects)

| المكتبة | الغرض | الرابط |
|---------|-------|--------|
| **Three.js** | مشاهد ثلاثية الأبعاد كاملة | [threejs.org](https://threejs.org/) |
| **angular-three** | غلاف Angular لـ Three.js | [npm](https://www.npmjs.com/package/angular-three) |
| **CSS 3D Transforms** | تأثيرات 3D بسيطة بـ CSS — بدون مكتبة إضافية | مدمج في CSS |
| **GSAP ScrollTrigger** | تأثيرات 3D مرتبطة بالتمرير | [gsap.com](https://gsap.com/docs/v3/Plugins/ScrollTrigger/) |

> **التوصية:** استخدام **CSS 3D Transforms** للتأثيرات البسيطة (قلب البطاقات، عمق الظلال) و **Three.js** فقط إذا احتجنا مشاهد 3D حقيقية.

### 4. الألوان والثيمات

| المكتبة | الغرض | الرابط |
|---------|-------|--------|
| **ngx-color-picker** | منتقي ألوان متكامل | [npm](https://www.npmjs.com/package/ngx-color-picker) |
| **Angular Material Theming** | نظام ثيمات ديناميكي | مضمن في Angular Material |
| **CSS Custom Properties** | متغيرات CSS للتخصيص الفوري | مدمج في CSS |

> **التوصية:** استخدام **ngx-color-picker** لاختيار الألوان و **CSS Custom Properties** لتطبيقها فوراً على العناصر.

### 5. الرسوم البيانية

| المكتبة | الغرض | الرابط |
|---------|-------|--------|
| **Chart.js + ng2-charts** | رسوم بيانية تفاعلية | [npm](https://www.npmjs.com/package/ng2-charts) |
| **ngx-echarts** | رسوم بيانية متقدمة مع تأثيرات | [npm](https://www.npmjs.com/package/ngx-echarts) |

> **التوصية:** **Chart.js + ng2-charts** للبساطة، أو **ngx-echarts** إذا نبي تأثيرات بصرية أقوى.

## البنية في قاعدة البيانات

### الجداول الموجودة

| الجدول | الوصف |
|--------|-------|
| `sidebar_sections` | أقسام القائمة الجانبية |
| `sidebar_items` | الشاشات في القائمة |
| `user_sidebar_config` | تخصيص لكل مستخدم |

### جداول جديدة مطلوبة

**جدول `screen_templates` (قوالب الشاشات):**

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | serial | المعرف |
| business_id | integer | ربط بالعمل |
| name | varchar(200) | اسم قالب الشاشة |
| description | text | وصف القالب |
| icon | varchar(50) | أيقونة |
| layout_config | jsonb | تخطيط العناصر (مواضع، أحجام) |
| is_system | boolean | قالب نظام (غير قابل للحذف) أو مخصص |
| is_active | boolean | نشط |

**جدول `screen_widgets` (عناصر الشاشة):**

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | serial | المعرف |
| screen_id | integer | ربط بـ sidebar_items (الشاشة) |
| widget_type | varchar(50) | نوع العنصر (templates/log/accounts/stats/chart/notes) |
| title | varchar(200) | عنوان العنصر |
| config | jsonb | إعدادات العنصر (ألوان، فلاتر، حسابات مختارة) |
| position_x | integer | موضع أفقي في الشبكة |
| position_y | integer | موضع عمودي |
| width | integer | عرض العنصر (عدد أعمدة) |
| height | integer | ارتفاع العنصر (عدد صفوف) |
| sort_order | integer | الترتيب |
| is_visible | boolean | ظاهر أو مخفي |

**جدول `screen_widget_templates` (ربط عنصر القوالب بالقوالب):**

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | serial | المعرف |
| widget_id | integer | ربط بـ screen_widgets |
| operation_type_id | integer | ربط بـ operation_types (القالب) |
| sort_order | integer | ترتيب القالب داخل العنصر |

**جدول `screen_widget_accounts` (ربط عنصر المراقبة بالحسابات):**

| الحقل | النوع | الوصف |
|-------|-------|-------|
| id | serial | المعرف |
| widget_id | integer | ربط بـ screen_widgets |
| account_id | integer | الحساب المراقب |
| sort_order | integer | الترتيب |

---

## العلاقة مع أنواع العمليات

```
صفحة أنواع العمليات → إنشاء القوالب (أنواع + قوالب + حسابات مرتبطة)
         ↓
الشاشات المخصصة → اختيار القوالب وتنفيذ العمليات عبر العناصر (Widgets)
```
