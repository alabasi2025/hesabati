# مواصفات الشاشة المخصصة "الحقيقية" (مثل التحصيل والتوريد)

## 1. الفكرة في سطر

- شاشة ثابتة بنفس هيكل صفحة التحصيل والتوريد.
- أسماء الأقسام/التبويبات قابلة للتعديل.
- لكل تبويب عمليات: مجموعة قوالب (أنواع عمليات) يختارها المستخدم عند الإنشاء.
- السجل واحد ثابت يعرض حركات الشاشة.
- قسم مراقبة: اسم قابل للتعديل + قائمة حسابات/صناديق يختارها المستخدم (صناديق، محفظة، صراف، إلخ).

---

## 2. قاعدة البيانات

### 2.1 جدول الشاشات (موجود)

`screen_templates`: نفس الجدول الحالي مع استخدام حقل يحدد نوع الشاشة:

- **template_key**: مثل `'collection_style'` للشاشات التي تعمل بهذا الهيكل (تحصيل/صرف/سجل/مراقبة).

### 2.2 إعداد الشاشة (تسميات + محتوى الخانات)

جدول يخزّن إعداد هذه الشاشة فقط (شاشة واحدة = سجل واحد):

**`screen_collection_style_config`** (أو custom_screen_config)

| الحقل | النوع | الوصف |
|-------|--------|--------|
| id | serial PK | |
| screen_id | integer FK → screen_templates.id | واحد لواحد |
| **التبويب الأول** | | |
| tab1_label | varchar(200) | مثلاً "عمليات القبض" |
| tab1_operation_type_ids | jsonb | [1, 3, 5] قوالب هذا التبويب |
| **التبويب الثاني** | | |
| tab2_label | varchar(200) | مثلاً "عمليات الصرف" |
| tab2_operation_type_ids | jsonb | [2, 4] قوالب التبويب الثاني |
| **قسم المراقبة** | | |
| accounts_section_label | varchar(200) | "صناديق" أو "محافظ" أو "بنوك"... |
| account_ids | jsonb | [10, 12, 15] حسابات/صناديق للعرض فقط |
| created_at, updated_at | timestamp | |

- **السجل**: لا يحتاج حقول إضافية؛ يعتمد على business_id + screen_id (أو مصدر الحركات) ويبقى واحداً ثابتاً في الواجهة.

> ملاحظة: إذا أردت دعم أكثر من تبويبين أو أكثر من قسم مراقبة واحد، يمكن استبدال الحقول أعلاه بجدول تفصيلي (مثلاً `custom_screen_tabs`: screen_id, order, label, operation_type_ids) وجدول `custom_screen_accounts`. المواصفة تبقى منطقياً نفسها.

---

## 3. الـ API

### 3.1 إعداد الشاشة (حفظ / جلب)

| Method | Endpoint | الوصف |
|--------|----------|--------|
| **GET** | `/businesses/:bizId/screens/:screenId/collection-style-config` | يرجع التسميات (tab1_label, tab2_label, accounts_section_label) + قوالب كل تبويب (tab1_operation_type_ids, tab2_operation_type_ids) + قائمة الحسابات (account_ids). إن لم يوجد سجل: قيم افتراضية ("تحصيل"، "توريد"، "الصناديق"، مصفوفات فارغة). |
| **PUT** | `/businesses/:bizId/screens/:screenId/collection-style-config` | Body: نفس الحقول أعلاه. يخزّن مرتبطاً بـ screen_id. |

### 3.2 قوالب وحسابات للقوائم

استخدام الـ endpoints الموجودة:

- **قوالب**: `GET /businesses/:bizId/operation-types` (مع فلتر category إذا لزم).
- **حسابات + صناديق**: `GET /businesses/:bizId/accounts?all=true` (أو ما يعيد الصناديق والمحافظ والصراف للاختيار).

### 3.3 السجل

استخدام endpoint سجل موجود (مثلاً widget-log أو journal-entries) مع تمرير screen_id أو business_id + فلتر تاريخ، ليعرض حركات العمليات المنفّذة من هذه الشاشة. لا حاجة لاختيار قوالب للسجل.

---

## 4. واجهة الإنشاء/التعديل (معالج أو صفحة إعداد)

عند إنشاء شاشة جديدة من نوع "مثل التحصيل والتوريد" (أو عند تعديل إعدادها):

| الخطوة | المحتوى | الحفظ |
|--------|---------|--------|
| **الخطوة 1 – التسميات** | حقل: اسم التبويب الأول (افتراضي "تحصيل") → tab1_label. حقل: اسم التبويب الثاني (افتراضي "توريد") → tab2_label. حقل: اسم قسم المراقبة (افتراضي "الصناديق") → accounts_section_label. | في الإعداد |
| **الخطوة 2 – قوالب التبويب الأول** | قائمة بأنواع العمليات (من operation-types). اختيار متعدد (checkboxes). | tab1_operation_type_ids |
| **الخطوة 3 – قوالب التبويب الثاني** | نفس الفكرة. | tab2_operation_type_ids |
| **الخطوة 4 – حسابات/صناديق قسم المراقبة** | قائمة بالحسابات والصناديق (من accounts)، مع فلتر نوع إن وُجد. اختيار متعدد. | account_ids |

(السجل لا يظهر في المعالج؛ يبقى ثابتاً في واجهة العرض.)

---

## 5. واجهة العرض (عند فتح الشاشة المخصصة)

- **المسار**: مثلاً `/biz/:bizId/custom-screens?screen=:screenId` يفتح شاشة واحدة بالهيكل الثابت.
- **جلب الإعداد**: استدعاء `GET .../screens/:screenId/collection-style-config` مرة واحدة.

**الرسم:**

- **تبويبان في الأعلى:**
  - التبويب الأول: العنوان من tab1_label، والأزرار/البطاقات من القوالب ذات الـ ids في tab1_operation_type_ids فقط.
  - التبويب الثاني: من tab2_label و tab2_operation_type_ids.
- عند الضغط على قالب: فتح نفس نموذج تنفيذ العملية الحالي (مبلغ، حساب، تاريخ...) ثم الحفظ؛ السجل يتحدث تلقائياً.
- **السجل**: منطقة ثابتة واحدة تحت التبويبات، تعرض سجل العمليات (بدون اختيار قوالب).
- **قسم المراقبة**: عنوان من accounts_section_label، وتحتيه قائمة أرصدة (وربما آخر حركات) للحسابات/الصناديق ذات الـ ids في account_ids فقط.

---

## 6. ملخص الربط

| العنصر | في DB | في API | في واجهة الإنشاء | في واجهة العرض |
|--------|--------|--------|-------------------|-----------------|
| اسم التبويب 1 | tab1_label | GET/PUT config | حقل نص | عنوان التبويب الأول |
| قوالب التبويب 1 | tab1_operation_type_ids | نفس الـ config | اختيار متعدد من operation-types | أزرار/بطاقات هذه القوالب فقط |
| اسم التبويب 2 | tab2_label | نفس الـ config | حقل نص | عنوان التبويب الثاني |
| قوالب التبويب 2 | tab2_operation_type_ids | نفس الـ config | اختيار متعدد | أزرار/بطاقات هذه القوالب فقط |
| السجل | - | سجل موجود | لا يظهر | منطقة سجل ثابتة |
| اسم قسم المراقبة | accounts_section_label | نفس الـ config | حقل نص | عنوان القسم |
| حسابات المراقبة | account_ids | نفس الـ config | اختيار متعدد من accounts/funds | قائمة أرصدة لهذه الحسابات فقط |

بهذا الشكل تكون الشاشة بعد الإنشاء ثابتة وحقيقية مثل التحصيل والتوريد، مع تسمياتك وقوالب كل خانة وحسابات المراقبة التي اخترتها، والسجل يبقى كما هو.

---

## 7. التنفيذ في المشروع

- **الجدول**: `screen_collection_style_config` (migration: `drizzle/0003_screen_collection_style_config.sql`).
- **الشاشات**: `screen_templates.template_key = 'collection_style'`.
- **API**: 
  - `GET/PUT /businesses/:bizId/screens/:screenId/collection-style-config` (camelCase: tab1Label, tab1OperationTypeIds, ...).
  - الحسابات الكاملة للاختيار: `GET /businesses/:bizId/accounts?all=true` → `{ accounts, stations }`.
  - السجل: `GET /businesses/:bizId/widget-log?dateFrom=...&dateTo=...&operationTypeIds=1,2,3` لتصفية حركات أنواع عمليات التبويبين فقط.
- **الواجهة**: معالج إنشاء 4 خطوات (التسميات → قوالب التبويب 1 → قوالب التبويب 2 → حسابات المراقبة + إضافة للقائمة). عرض الشاشة: تبويبان + سجل + مراقبة مع زر "إعداد الشاشة" لتعديل الإعداد لاحقاً.
