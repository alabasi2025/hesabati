# آلية الترقيم في النظام المحاسبي

## 📋 المبدأ الأساسي

آلية الترقيم في النظام تعتمد على **أنواع الحسابات الفرعية (Account Sub-Natures)** وليس على التصنيفات.

---

## 🏗️ البنية الأساسية

### 1️⃣ أنواع الحسابات الفرعية (account_sub_natures)
هي الأنواع **النظامية** التي تحدد طبيعة الحساب وتؤثر على العمليات والترقيم:

| النوع الفرعي | المفتاح | البادئة | مثال الكود |
|--------------|---------|---------|------------|
| صندوق | `fund` | `FND` | `FND-01`, `FND-02` |
| بنك | `bank` | `BNK` | `BNK-01`, `BNK-02` |
| محفظة إلكترونية | `e_wallet` | `WLT` | `WLT-01`, `WLT-02` |
| صراف | `exchange` | `EXC` | `EXC-01`, `EXC-02` |
| مخزن | `warehouse` | `WHS` | `WHS-01`, `WHS-02` |
| مورد | `supplier` | `SUP` | `SUP-01`, `SUP-02` |
| موظف | `employee` | `EMP` | `EMP-01`, `EMP-02` |
| شريك | `partner` | `PRT` | `PRT-01`, `PRT-02` |

### 2️⃣ التصنيفات (Classifications)
التصنيفات مثل `fund_types`, `bank_types`, `warehouse_types` تُستخدم **للتنظيم والفلترة فقط**:

- ✅ **هدفها:** تنظيم العناصر في الواجهة وتسهيل البحث
- ❌ **ليس لها علاقة بالترقيم:** الترقيم يعتمد على النوع الفرعي فقط

**مثال:**
- صندوق 1: تصنيف "صندوق نقدي" → الكود: `FND-01`
- صندوق 2: تصنيف "صندوق المبيعات" → الكود: `FND-02`
- صندوق 3: تصنيف "صندوق نقدي" → الكود: `FND-03`

## 📊 آلية الترقيم للحسابات

### 🌳 الحسابات الرئيسية (isLeafAccount = false)
- **الهدف:** التنظيم الهرمي فقط
- **آلية الترقيم:** شجرية `parent.child`
- **أمثلة:** `1`, `1.1`, `1.2`, `2`, `2.1.1`
- **النوع الفرعي:** لا يوجد

```typescript
// مثال:
الحساب الرئيسي: "الأصول" → الكود: 1
  ├─ الحساب الفرعي: "الأصول المتداولة" → الكود: 1.1
  └─ الحساب الفرعي: "الأصول الثابتة" → الكود: 1.2
```

### 🍃 الحسابات الفرعية (isLeafAccount = true)
- **الهدف:** الحسابات المالية الفعلية التي تتأثر بالعمليات
- **آلية الترقيم:** حسب النوع الفرعي
- **النوع الفرعي:** مطلوب

```typescript
// أمثلة:
حساب صندوق → النوع الفرعي: fund → الكود: FND-01
حساب بنك → النوع الفرعي: bank → الكود: BNK-01
حساب محفظة → النوع الفرعي: e_wallet → الكود: WLT-01
```

## 🔧 التطبيق التقني

### في ملف `sequencing.ts`

```typescript
// للحسابات الفرعية
export async function generateLeafAccountCode(
  businessId: number,
  natureKey: string,
  tx?: DbOrTx,
): Promise<{ code: string; sequenceNumber: number }> {
  const counterType = `account_${natureKey}`;
  const sequenceNumber = await getNextSequence(businessId, counterType, 0, 0, tx);
  const prefix = TYPE_PREFIXES[natureKey] || natureKey.toUpperCase().substring(0, 3);
  const code = `${prefix}-${String(sequenceNumber).padStart(2, "0")}`;
  return { code, sequenceNumber };
}

// للحسابات الرئيسية
export function generateTreeAccountCode(
  parentCode: string | null,
  sequenceNumber: number,
): string {
  if (!parentCode) return String(sequenceNumber);
  return `${parentCode}.${sequenceNumber}`;
}
```

### في ملف `accounts.routes.ts`

```typescript
// عند إنشاء حساب جديد
if (isLeafAccount && subNature) {
  // حساب فرعي: استخدم آلية الكود حسب النوع الفرعي
  const result = await generateLeafAccountCode(bizId, subNature.natureKey, db);
  sequenceNumber = result.sequenceNumber;
  generatedCode = result.code; // FND-01, BNK-01, إلخ
} else {
  // حساب رئيسي: استخدم آلية الترقيم الشجري
  sequenceNumber = await getNextChildAccountSequence(bizId, parentAccountId, db);
  generatedCode = generateTreeAccountCode(parent?.code ?? null, sequenceNumber); // 1, 1.1, إلخ
}
```

## ✅ ملخص القواعد

| العنصر | النوع الفرعي | التصنيف | الكود | الاستخدام |
|--------|--------------|---------|-------|-----------|
| **صندوق** | `fund` | للتنظيم فقط | `FND-01` | عمليات نقدية |
| **بنك** | `bank` | للتنظيم فقط | `BNK-01` | عمليات بنكية |
| **محفظة** | `e_wallet` | للتنظيم فقط | `WLT-01` | عمليات محافظ |
| **صراف** | `exchange` | للتنظيم فقط | `EXC-01` | عمليات صرافة |
| **مخزن** | `warehouse` | للتنظيم فقط | `WHS-01` | عمليات مخزنية |
| **حساب رئيسي** | لا يوجد | - | `1`, `1.1` | للتنظيم فقط |

---

## 🎯 الخلاصة

1. **أنواع الحسابات الفرعية** (account_sub_natures) هي الأساس وتحدد الكود
2. **التصنيفات** (fund_types, bank_types, إلخ) للتنظيم والفلترة فقط
3. **الحسابات الفرعية** تأخذ كود حسب النوع (FND-01, BNK-01)
4. **الحسابات الرئيسية** تأخذ كود شجري (1, 1.1, 2.3)
5. كل عمل/منشأة له نظام ترقيم مستقل تماماً
